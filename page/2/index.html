Docker Study Notes

Introduction

A platform for building, running and shipping applications in a consistent manner

  • It solves the problem that when you develop an application works in your development environment, it doesn’t not work in the production environment.
    • One or more files missing
    • Software version mismatch
    • Different configuration settings

Docker will package everything needed to run the application.

Docker will run all the required application in an insulated environment called container.

Those containers allows multiple applications use different versions of some software side by side

During the removal, User can simply remove all one project with all the dependence without affect another one.

Container VS Virtual Machine

Problem of VM

  1. Each VM needs a full-blown OS
  2. Slow to start
  3. Resource Intensive

Benefit of Containers

  1. Allow running multiple apps in isolation
  2. Are lightweight
  3. Use OS of the host
  4. Start quickly
  5. Need less hardware resources

Docker Architecture

Use a client-server architecture

Use rest API

The server is called Docker Engine

A container is just a computer process, share the kernel of the host

  • Windows can run windows+linux container
  • Linux can run linux container
  • Mac can run Linux based on Linux VMB

Package

A Dockerfile will package everything our application needs to into an image

Image:

  • A cut-down OS
  • A runtime environment (eg Node)
  • Application files
  • Third-party libraries
  • Environment variables

CrowdStrike学习笔记

因为工作需求，最近学习了一款叫做CrowdStrike的安全管理软件。这款软件本身是具有一定特殊性的，它的公司Falcon是一家并不在并且也不准备在中国进行销售的公司，所以本质上应该所有本土的中国公司并不会采购这款软件并且进行使用。只有像是一些跨国企业的才会有可能去使用它。

不同于传统意义上的那些杀毒防护安全软件，CrowdStrike本质上有着和它们明显的区别。传统意义上的杀毒软件会定期对整台电脑里面的每个文件进行杀毒，杀毒的方式是通过对比本地杀毒软件的病毒特征数据库以及本地文件的特征来进行判断。当这样的文件被执行时，杀毒软件间会进行组织并且进行隔离，等待使用者的下一步处理。这样做的好处是比较全面，但是同时也有着不少缺点，如：

  1. 杀毒软件使用时是十分重量级别的，会消耗很多电脑的资源从而导致电脑变得卡顿，死机或者增加笔记本的电量消耗（风扇呜呜转）
  2. 因为是在本地进行查杀，对于一些最新的zero day漏洞没有办法进行非常及时的处理
  3. 功能可能比较单一，可能只能做到杀毒以及一些基本的防护。

CrowdStrike这款软件非常好的解决了上面的这些特点。首先它是一款十分轻量级别的产品。客户终端安装这个软件之后，它不会对每个本地文件进行定情的全盘查杀。取而代之的是，它会在程序被执行时，对每个执行的程序，子程序进行验证从而来决定是否可以执行以及对管理员进行报警。它不会对终端用户的所安装的软件进行控制，只会对终端安装的软件开始运行时进行阻止，并且可以细致到子程序。

因为总有一些特殊的原因，企业并没有办法阻止用户去安装一些盗版软件。而盗版软件有时候的做法是在正常的软件中夹杂一些恶意程序（广告程序之类的）。CrowdStrike是可以在这样的盗版软件运行时，只阻止这些恶意子程序，剩下的正常程序还是可以正常运行。似乎可以名正言顺的用盗版了。

除了在防范方面，CrowdStrike还是一个非常整体的安全管理软件。经过一段日子的简单学习，我对于它在日常以及在事件发生时有着以下的理解。可能不全面，等过一些日子用久了之后再进行更新

日常防范 - 日志以及detection处理

Activity → Dashboard

一个操作平台的总览，里面会详细介绍所有已经检测到的detection以及所有防御的事件

Activity → Detections

每天进行所有被CrowdStrike sensors检测到的危险漏洞，并且进行认知以及处理。

事件方法 - 对突发事件的处理

Activity → Incidents

当有些事件被定性之后，就会上升到incidents。 Incidents通常会和detection连在一起。如果detection那边查到的漏洞是导致incident的原因，那detection那边就会直接现实到incident的链接，方便追踪

Activity → Quarantied Files

当detection中出现了被隔离的文件，可以通过这个功能来查看并且信任放行可以文件。这个信任放行文件是可以重做的

Activity → Real Time Response

实时回应事件是可以通过Falcon控制台来进行查看的。这项权力一般性只在少数人手里。有这项权利的人可以远程操控客户终端的一些行为并且进行人为干预，如停止某个线程的进行以及远程删除非法程序。

Activity → Firewall Events

可以查看被防火墙过滤的事件详情，以方便判断归纳。如果同一个事件长期频繁出现并且调查后并没有什么问题，可以考虑加入白名单

Activity → USB DEVICE CONTROL

可以查看客户终端的USB行为

综上，运维需要用到的大概是两个方面：

  1. 日常维护的时候，每天定时（早上和下午）查看detection。对每一个detection进行定性，如果没法判断就上报。如果成为incident的，就得立即展开调查。
  2. 如果遇到客户有突发事件，第一要立刻断网，第二对目标终端进行分析。通过对特定host的日志进行提取(CrowdStrike的一个好处就是可以记录终端的用户行为日志），并且结合使用者的活动来对可以的日志进行深一步的调查。
